Defining Bindings

Currently, only one format is supported for bindings definitions: SIDL, an SGML-esque Interface Definition Language. However, the generator was designed to be flexible enough to support multiple formats, and there are plans to define a Tab-delimited Interface Definition Language as well.

There are currently four pages in this section:

  • Bindings
  • Binding
  • Functions
  • Types

Note: Many of the objects described here can take a doc attribute. This element is used to generate inline documentation for the object (but this feature is not fully implemented yet). Although conceptually doc is an attribute of the object, in the SIDL format it is implemented as a child element, that is:

<object>
    <doc>...</doc>
    ...
</object>

and not:

<object doc="...">
    ...
</object>
