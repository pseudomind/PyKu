Bindings

A Bindings object represents a set of bindings that are to be generated together.

Attributes

  • name: (required) the name for this binding; it will be generated using this name.
  • version: (required) the version number for this binding.
  • doc: a documentation string.

Children

  • import: (zero or more) other bindings definitions to be absorbed into this one.
  • bundle: (zero or more) other bindings definitions to be bundled with this one.
  • include: (zero or more) header files to be included when generating the binding.
  • link: (zero or more) libraries to link against generating the binding.
  • types: (zero or more) types used in the C++ code.
  • binding: (one or more) C++ objects/namespaces to bind.

Import

An Import object represents additional Bindings objects that are to be imported. The contents of the imported files are treated as though they were defined in the current file.

Attributes

none

Children

  • file: (zero or more) files to import.

Bundle

A Bundle object represents additional Bindings objects that are to be generated together with the current one.

Attributes

none

Children

  • file: (zero or more) files to import.

Include

An Include object represents header files to include when generating the bindings. Currently only the angle-bracket form (#include <...>) is supported, since the original target for the generator (the Haiku API) doesn't require the quoted form (#include "...").

Note: Even though this is intended for including headers files, once support for the quoted form is implemented, it would be possible to use it to include other file as well.

Attributes

none

Children

  • lib: (zero or more) files to include.

Link

A Link object represents libraries to be linked against when generating the bindings. Currently only runtime-binding is supported, since the original target for the generator (the Haiku API) doesn't require static binding.

Attributes

none

Children

  • lib: (zero or more) libraries to link against.

File

A File object represents a file on the file system. The precise interpretation of the filename depends on the parent object of the File.

Attributes

  • name: (required) the filename.

Children

none

Lib

A Lib object represents a library to link against. This should be in the form required by the linker.

Attributes

  • name: (required) the libname.

Children

none
