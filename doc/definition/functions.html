Functions

A Functions object represents a set of class/object methods or namespace functions to expose.

Attributes

none

Children

  • constructor: an object constructor.
  • destructor: an object destructor.
  • method: an object method.
  • event: an object method that will be called from C++, i.e., in response to a user event.
  • static: a class method.
  • plain: a plain function.

<Function>

<Function> is not a single type, but one of several types. A <Function> represents a function or method to expose. Supported function types are:

  • constructor
  • destructor
  • method
  • event
  • static
  • plain

Attributes

  • name: (required) the function name (not used for constructors or destructors).
  • overload-name: when target language does not support overloaded functions, it will use this name to disambiguate (not used for destructors).
  • doc: a documentation string.
  • type
  • string-length
  • max-string-length
  • array-length
  • max-array-length
  • pass-as-pointer

Children

  • param: a function parameter.
  • return: a return value.

Param/Return

A Param object represents a function parameter; a Return object represents a return value. Since C++ and the target language do not have the same syntax, each Param and Return has an associated action. Supported actions are:

  • input: should be treated as target-language input
  • output: should be treated as an output if the target language handles multiple outputs; otherwise should be treated as input
  • error: should be treated as an error; the target language should set an error value or raise an exception
  • length[X]: should be generated from the length of X if the target language stores the length with the variable; otherwise should be treated as input
  • count[X]: should be generated from the count of X if the target language stores the count with the variable; otherwise should be treated as input

Note that length[X] and count[X] are simply indications that a C++ variable needs to be generated to hold the value. They do not need to be used if the C++ variable already exists (i.e., as a constant), since the X parameter itself also contains the necessary length and count information.

Attributes

  • name: (required for Param) the parameter name (used for keyword-style entry).
  • action: defaults to input for Param and output for Return.
  • success: used with the error action to indicate the success value.
  • type
  • must-not-delete
  • string-length
  • max-string-length
  • array-length
  • max-array-length
  • pass-as-pointer

Children

none
